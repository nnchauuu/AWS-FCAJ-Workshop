## Event Information

- **Event Name:** FCAJ Community Day - June 2026

- **Format:** In-person attendance
- **Location:** FCAJ Community Day event venue
- **Role:** Attendee

---

# Event Objectives

FCAJ Community Day - June 2026 was organized to create an opportunity for students, developers, and people interested in technology to connect and learn practical experiences from speakers in the fields of AWS, AI, DevOps, Cloud Architecture, and technology product development.

The event aimed to achieve the following goals:

- Introduce current trends in applying AI to real-world products.
- Share experience in building AI Voice Agents capable of natural conversations.
- Explore the challenges of deploying AI systems into production environments.
- Study how tools, data, and enterprise systems can be connected through MCP.
- Provide practical perspectives on AWS, Cloud Architecture, DevOps, and system operations.
- Inspire attendees to keep learning, practicing technology, and developing product thinking.

---

# Session 1: Voice Agents - Building Human-Like AI Conversations at Scale

This session focused on building AI Voice Agents that can communicate naturally with users, especially in Vietnamese-language contexts and systems that require real-time responses.

### Key Topics

- Overview of AI Voice Agents and their role in modern technology products.
- Analysis of the conversation processing flow, including Speech-to-Text, Large Language Models, and Text-to-Speech.
- Challenges in building natural Vietnamese conversations.
- The importance of latency when the system needs to respond in real time.
- Experience handling interruptions, context, emotion, and tone in conversations.
- How Voice Agents can connect with tools to perform business actions.

### Key Takeaways

- Building an AI Voice Agent is not only about making a model speak, but also about ensuring natural, contextual, and fast conversations.
- Latency is a critical factor that directly affects user experience.
- Production AI systems need to balance response quality, infrastructure cost, and scalability.
- Vietnamese real-time conversations introduce challenges in tone, context, and response speed.

---

# Session 2: Building Secure Private MCP Connection

This session introduced MCP and how to build private, secure connections between AI Agents and internal tools or systems.

### Key Topics

- Introduction to MCP and its role in the AI Agent ecosystem.
- How AI Agents can connect to external tools to perform real-world tasks.
- Security requirements when connecting AI with enterprise data and systems.
- Experience building private MCP connections to reduce the risk of data leakage.
- How MCP can be combined with AWS services in system architecture.
- The role of authorization, access control, and data protection when deploying AI Agents.

### Key Takeaways

- MCP allows AI Agents to do more than answer questions; they can interact with tools and perform actions.
- Security and authorization are essential when connecting AI to real systems.
- Private connections help reduce risks when working with internal data.
- AI Agents must be designed with proper controls to avoid incorrect data access or unintended actions.

---

# Session 3: AWS, AI, DevOps, and Cloud Architecture

This session provided an overview of how AWS, AI, and DevOps can be combined to build modern systems that are scalable and easier to operate.

### Key Topics

- The role of AWS in deploying modern AI and backend systems.
- How Cloud Architecture supports scaling when user traffic grows.
- The importance of DevOps in automating deployment and operations.
- Important considerations when bringing AI models into production.
- Experience balancing performance, cost, and system stability.
- How to monitor, test, and improve systems after deployment.

### Key Takeaways

- AWS provides many services for deployment, scaling, and system monitoring.
- DevOps helps software development and deployment become faster and more stable.
- Cloud Architecture should be designed based on real needs instead of adding unnecessary complexity.
- AI deployment requires attention to latency, security, cost, and scalability.
