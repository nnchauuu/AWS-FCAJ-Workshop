# Sharing and Feedback

## General Evaluation

### 1. Working Environment

The working environment in the internship program at FCAJ was quite comfortable, friendly, and supportive for interns to learn and improve. The flexible working format allowed me to manage my time between studying, practicing labs, and participating in the group project.

During the internship, I noticed that everyone was always willing to support each other when difficulties occurred, especially in tasks related to configuring AWS services, deploying applications to the cloud, and troubleshooting issues between the frontend, backend, and services such as EC2, S3, SES, CloudWatch, Cognito, ALB, and Auto Scaling. Thanks to this, I became more confident when approaching new technologies.

I think the program could add more periodic discussion sessions among internship groups so that everyone can share progress, difficulties, and practical project implementation experience.

### 2. Support from Mentors / Admin Team

The mentors and admin team provided enthusiastic support throughout the internship. They gave clear guidance on the learning direction, the topics to study, and how to approach AWS services step by step.

In particular, during the project implementation process, the mentors not only explained how to configure services but also helped me understand why each service should be used. For example, EC2 was used to deploy the backend, S3 was used for file storage, SES was used for sending emails, CloudWatch was used for system monitoring, ALB and Auto Scaling were used to improve scalability, and Cognito was used to support user authentication.

What I appreciated most was that the mentors encouraged interns to research, analyze errors, and propose solutions before receiving detailed instructions. This approach helped me improve my self-learning ability and problem-solving mindset.

### 3. Relevance Between Tasks and Major

The internship contents were very suitable for my major in Information Technology. Knowledge related to web programming, databases, security, system deployment, and cloud computing could be directly applied during the project implementation process.

During the internship, I had the opportunity to connect the theoretical knowledge I had learned at school with real deployment experience on AWS. Tasks such as configuring Google/Cognito login, sending emails with SES, uploading files to S3, monitoring resources with CloudWatch, deploying the backend to EC2, configuring CloudFront for the frontend, and learning about VPC architecture helped me better understand how a web system operates in a cloud environment.

As a result, I not only understood the theory better but also gained practical experience in implementation, troubleshooting, and evaluating system architecture in real-world scenarios.

### 4. Learning Opportunities and Skill Development

Participating in the program helped me gain a lot of new knowledge about AWS and real application deployment processes. Some of the topics I learned and practiced include:

- Learning and using AWS services such as IAM, EC2, S3, RDS, CloudFront, CloudWatch, SES, Cognito, ALB, Auto Scaling, and VPC.
- Deploying a web application to a cloud environment instead of only running it locally.
- Configuring Amazon SES to send emails for features such as notifications, confirmations, and reminders.
- Using Amazon S3 to store files, images, and static resources.
- Learning how to monitor a system with CloudWatch through metrics such as CPU, RAM, and application logs.
- Becoming familiar with real deployment architecture, including frontend, backend, database, load balancer, and supporting services.
- Practicing how to read documentation, debug errors independently, and document deployment procedures.

Besides technical skills, I also improved my teamwork, communication, progress reporting, and ability to proactively discuss problems during the project implementation process.

### 5. Culture and Teamwork Spirit

The working spirit in the program was quite positive. Team members coordinated, supported, and shared work with each other during the project implementation process. When errors or difficulties occurred during configuration, everyone could discuss together to find a suitable solution.

The working environment did not create unnecessary pressure, but it still helped interns become responsible for their assigned tasks. This made me feel more comfortable during the process of learning, experimenting, and improving my skills.

Through teamwork, I better understood the importance of task division, progress updates, and agreeing on the system architecture before deployment.

### 6. Policies and Benefits for Interns

The program created good conditions for interns by providing learning materials, practical labs, and opportunities to communicate with mentors. Being able to participate in a real project was a valuable opportunity for me to apply what I had learned to a specific product.

For me, the greatest value of the program was not only learning more AWS services, but also understanding the process of building and deploying a web system on the cloud from beginning to end. This was a very useful experience for my studies, academic projects, and future career orientation.

---

## Additional Questions

### What I Was Most Satisfied With

What I was most satisfied with was the opportunity to access and practice many AWS services in a real project. Previously, I mainly studied theory or deployed applications in a local environment. After the internship, I understood more clearly how to deploy a web system to the cloud, how different services work together, and how to handle errors that occur during deployment.

In particular, tasks such as configuring EC2, S3, SES, CloudWatch, Cognito, CloudFront, ALB, and Auto Scaling gave me a more practical understanding of system architecture on AWS.

### What I Think Could Be Improved

The program could add more in-depth training sessions on topics such as AWS security, practical cloud architecture design, cost optimization, CI/CD processes, and common troubleshooting methods when deploying applications.

In addition, if there were more product demo sessions or periodic progress review sessions between groups, interns could learn more experience from each other’s projects.

### Would I Recommend This Program to My Friends?

Yes. I would recommend this program to friends who are interested in Cloud, DevOps, backend development, or system deployment. The program helps interns access practical knowledge, receive support from mentors, and gain hands-on experience with AWS services through labs and projects.

This is a suitable environment for students to improve their professional skills, self-learning ability, and familiarity with real-world application deployment processes.

---

## Suggestions and Expectations

I hope the program can organize more in-depth AWS workshops, especially on topics such as VPC, system security, backend production deployment, CI/CD, monitoring, and cost optimization.

If the program offers more advanced activities or opportunities to continue developing projects after the internship, I would be very willing to participate in order to keep learning, improve my skills, and gain more practical experience.
